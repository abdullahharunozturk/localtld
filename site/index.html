01How it works

No magic — it orchestrates two standard tools: dnsmasq for DNS and Caddy as a reverse proxy.

browser → panel.aaron.localtld │ dnsmasq: *.localtld → 127.0.0.1 (+ macOS /etc/resolver) ▼ 127.0.0.1:80 → Caddy (Host header → the right port) ▼ 127.0.0.1:63469 ← your dev server (localtld assigned PORT)

02Why it's nice

Dynamic ports

Run hundreds of projects at once. No fixed ports, no collisions, nothing to remember.

Zero-config

One line in package.json. Wire it once; every project just works.

Graceful fallback

On a machine without localtld, projects still run on localhost:PORT. Nobody's blocked.

Safe by default

.localtld is never a real TLD. Pick a real one and it warns you first.

05Your TLD, your call

Default is .localtld. Shorten or brand it anytime:

$ localtld tld test # panel.aaron.test
$ localtld tld localtld.sh # panel.aaron.localtld.sh

Heads up. If you pick a real TLD (.com, .dev…), every site under it resolves to 127.0.0.1 on your machine. localtld checks against the IANA root zone and asks you to confirm — so you never do it by accident.
